# 01 项目总方案

## 1. 项目概述

### 1.1 项目名称
销售采购一体化智能体平台（Revenue & Procurement Copilot Platform，简称 RPCP）

### 1.2 项目背景
企业在销售与采购两大关键经营链路中，普遍存在信息分散、流程割裂、经验依赖、决策滞后、知识无法复用的问题。

销售侧典型痛点：
- 线索来源复杂，质量参差不齐
- 高价值客户识别依赖资深销售经验
- 跟进记录不完整，线索流转不可控，丢单严重
- 招投标和解决方案编写效率低，跨部门协同成本高
- 历史方案、案例、产品能力材料难以复用

采购侧典型痛点：
- 供应商信息分散在 ERP、SRM、Excel、邮件、质检系统、投诉系统中
- 供应商评价标准不统一，结果主观性强
- 采购询价流程繁琐，报价对比结构化程度低
- 历史询价经验与采购知识难沉淀、难复用
- 风险识别滞后，缺乏全局供应商画像

在 AI 大模型、知识检索、流程自动化、企业系统集成能力成熟的背景下，建设一套统一的企业级智能体平台，已具备显著的业务价值与技术可行性。

### 1.3 项目目标
建设一个覆盖销售与采购核心场景的企业级智能体平台，实现：

1. 销售提效：
   - 自动完成线索洞察、评分、优先级排序、跟进建议生成
   - 自动生成投标方案、售前方案初稿、商务建议与交付建议

2. 采购提效：
   - 自动汇聚供应商多源数据，生成客观、可解释的供应商评分与合作建议
   - 自动生成询价文档、报价结构化对比表、采购建议

3. 管理提效：
   - 形成统一的数据底座、知识底座、评估底座、流程底座
   - 建立可审计、可回溯、可监控、可持续优化的企业级 AI 运营体系

4. 战略价值：
   - 让经验转化为平台能力
   - 让文档、数据、历史案例成为可调用资产
   - 为未来扩展到法务、财务、客服、供应链等领域奠定平台基础

## 2. 总体业务范围

### 2.1 纳入范围
本期统一纳入四大业务模块：

#### 模块 A：销售线索筛选与跟进智能体
核心能力：
- 客户画像抽取
- 线索评分
- 需求成熟度识别
- 跟进建议生成
- CRM 自动写入与更新
- 销售任务提醒与升级预警

#### 模块 B：招投标/方案生成智能体
核心能力：
- RFP/招标文件解析
- 客户需求结构化抽取
- 历史案例召回
- 产品能力映射
- 技术方案与商务方案协同生成
- 方案润色、审校、版本对比

#### 模块 C：供应商评估智能体
核心能力：
- 多源供应商数据整合
- 供应商评分模型
- 风险识别与预警
- 供应商横向对比分析
- 合作建议输出
- 分级分层管理建议

#### 模块 D：采购询价与比价智能体
核心能力：
- 采购需求抽取与标准化
- 询价函自动生成
- 报价单结构化解析
- 多供应商比价分析
- 历史采购经验复用
- 采购建议与决策辅助输出

### 2.2 不纳入范围
本期不作为首期核心交付范围：
- 全自动合同签署闭环
- 付款审批自动决策
- 财务总账自动入账
- 外部电商采购平台全流程替代
- 无人工审批的自动采购执行

## 3. 产品定位

### 3.1 产品愿景
打造面向中大型企业的“经营决策增强型 AI 平台”，让销售更快拿下正确客户，让采购更稳选对供应商。

### 3.2 产品定位一句话
以企业数据、知识、流程为基础，面向销售与采购核心岗位提供可执行、可解释、可审计、可集成的智能体服务平台。

### 3.3 目标客户画像
- 中大型制造业企业
- ToB 软件与集成服务企业
- 工程项目型企业
- 医疗、能源、汽车、电子、快消等拥有复杂采购与销售流程的集团企业
- 已部署 CRM、ERP、SRM、OA、知识库等系统，希望在其上叠加 AI 能力的企业

## 6. 关键业务流程概览

### 6.1 销售线索流程
线索接入 → 数据补全 → 客户画像生成 → 线索评分 → 优先级排序 → 跟进建议 → CRM 更新 → 跟进结果反馈 → 模型校准

### 6.2 方案生成流程
上传需求文档/RFP → 文档解析 → 需求结构化 → 案例/产品/交付知识召回 → 大纲生成 → 技术方案生成 → 商务方案生成 → 审校与合规检查 → 导出与协同评审

### 6.3 供应商评估流程
汇聚供应商多源数据 → 数据标准化 → 指标计算 → 风险识别 → 评分与分层 → 横向对比 → 合作建议生成 → 管理层审批/备案

### 6.4 采购询比价流程
录入采购需求 → 需求结构化 → 询价函生成 → 供应商邀约 → 报价收集 → 报价单解析 → 横向比较 → 风险与偏差说明 → 采购建议输出 → 归档沉淀

## 7. 企业级方案设计原则

1. 可上线：不是演示系统，而是生产级平台。
2. 可集成：适配 CRM、ERP、SRM、OA、邮件、文档系统、对象存储。
3. 可解释：评分、建议、推荐都必须给出原因与证据。
4. 可审计：所有模型输入、输出、人工干预、系统动作均需留痕。
5. 可治理：支持提示词治理、知识治理、模型路由治理、风险策略治理。
6. 可扩展：后续可扩展到合同审查、订单风险、客服辅助、供应链预测等场景。
7. 安全合规：满足企业在权限、数据脱敏、隐私保护、日志审计方面要求。
8. 人机协同：关键决策保留人工确认，不做不可控的全自动黑盒执行。

## 10. 建议实施策略

建议采用“平台先行、场景分期、双域联动、持续运营”的总体策略：

第一阶段：
优先落地销售线索筛选与采购询比价两个 ROI 明确、数据相对容易收集的模块。

第二阶段：
扩展招投标方案生成与供应商评估模块，形成较完整的销售采购闭环。

第三阶段：
沉淀知识运营、模型运营、反馈训练、经营分析驾驶舱，走向集团级平台化运营。

## 11. 结论

这是一个具备真实业务价值、可逐步落地、可形成企业核心竞争力的 AI 平台项目。其先进性不在于单一大模型能力，而在于将数据、知识、流程、规则、评估、安全、运营整合为统一体系。

真正达到业内顶尖水准的关键，不是“能生成内容”，而是：
- 能稳定产生高质量结果
- 能进入业务主流程
- 能被管理层信任
- 能持续优化与规模复制

本方案将按照这一标准进行设计与交付。
