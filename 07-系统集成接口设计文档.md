# 07 系统集成接口设计文档

## 1. 文档目标

本文档用于定义销售采购一体化智能体平台与企业内外部系统的集成边界、接口清单、交互方式、字段要求、安全机制、异常处理与上线规范，确保平台具备企业级集成落地能力。

## 2. 集成总原则

- 优先标准 API 集成，减少对源系统侵入
- 接口幂等、可重试、可审计
- 数据传输最小必要原则
- 关键链路必须有失败补偿机制
- 接口协议统一、认证统一、日志统一
- 重要主数据接口优先实时同步，非关键数据可采用准实时/批量

## 3. 集成对象清单

### 3.1 销售侧系统
- CRM
- 官网表单系统
- 邮件系统
- 企业 IM
- 客户数据库/外部企业信息服务

### 3.2 采购侧系统
- ERP
- SRM
- 采购申请系统
- 质检系统
- 投诉/客服系统
- 合同系统

### 3.3 公共支撑系统
- OA/流程审批系统
- 统一身份认证 SSO
- 知识库/文档系统
- 文件存储系统
- 数据平台/主数据平台
- 消息通知平台

## 4. 集成架构建议

建议采用“API Gateway + Integration Service + Event Bus + ETL/CDC”的混合集成模式：
- 实时查询与写回：走 API
- 事件通知：走消息队列/Webhook
- 大批量历史数据：走 ETL/批处理
- 主数据变更：走 CDC 或事件总线

## 5. 接口分类

- 查询类接口
- 写回类接口
- 上传类接口
- 回调类接口
- 批量同步类接口
- 字典/主数据类接口
- 审批触发类接口

## 6. 核心接口清单

## 6.1 CRM 接口

### 6.1.1 查询线索列表
- 接口名称：Get Lead List
- 方法：GET
- 路径：`/api/integration/crm/leads`
- 用途：获取待分析线索或增量线索

请求参数：
- page_no
- page_size
- updated_after
- owner_user_id
- status

返回字段：
- lead_id
- company_name
- contact_name
- industry
- source_channel
- status
- owner_user_id
- created_time
- updated_time

### 6.1.2 查询线索详情
- 方法：GET
- 路径：`/api/integration/crm/leads/{leadId}`

### 6.1.3 回写线索评分与建议
- 方法：POST
- 路径：`/api/integration/crm/leads/{leadId}/ai-insight`

请求体示例：
```json
{
  "aiLeadScore": 87,
  "aiMaturityLevel": "high",
  "aiPriorityLevel": "P1",
  "aiNextAction": "建议24小时内安排电话沟通并发送制造业案例包",
  "aiProfileSummary": "中大型制造企业，采购角色明确，需求成熟度较高",
  "aiRiskFlag": ["预算未明确"],
  "generatedAt": "2025-01-01T10:00:00Z"
}
```

### 6.1.4 写入跟进记录
- 方法：POST
- 路径：`/api/integration/crm/followups`

## 8. 标准返回规范

统一返回格式建议：
```json
{
  "code": "0",
  "message": "success",
  "requestId": "req_123456",
  "data": {}
}
```

错误返回示例：
```json
{
  "code": "CRM_401_001",
  "message": "unauthorized",
  "requestId": "req_123456",
  "data": null
}
```

## 9. 异常码规范

建议规则：
- 系统前缀 + HTTP 类别 + 业务序号

示例：
- CRM_400_001 参数缺失
- CRM_401_001 认证失败
- ERP_404_001 单据不存在
- SRM_409_001 数据版本冲突
- AI_500_001 模型服务异常

## 10. 认证与安全机制

### 10.1 认证方式
- OAuth2 Client Credentials
- JWT
- API Key（仅内部受控场景）
- mTLS（高安全等级接口）

### 10.2 安全要求
- 所有接口走 HTTPS
- 接口签名校验
- 请求时间戳防重放
- 白名单限制
- 敏感字段加密传输

## 11. 幂等与重试机制

### 11.1 幂等场景
以下接口必须支持幂等：
- 回写线索分析结果
- 回写供应商评估结果
- 发起审批
- 上传文档元数据登记
- 写入报价结构化结果

### 11.2 幂等键建议
- idempotency_key
- source_request_id
- business_id + operation_type

### 11.3 重试策略
- 网络抖动类错误自动重试
- 业务类错误不自动重试
- 重试次数建议 3 次
- 指数退避策略

## 12. 数据同步策略

### 12.1 实时同步
适用：
- 新线索进入
- 审批结果变更
- 关键任务状态更新

### 12.2 准实时同步
适用：
- 邮件沟通摘要
- 交付记录
- 投诉记录

### 12.3 批量同步
适用：
- 历史供应商数据
- 历史订单数据
- 旧方案与知识文档迁移

## 13. 日志与审计要求

每次接口调用建议记录：
- request_id
- source_system
- target_system
- operator_id
- api_name
- request_time
- response_time
- success_flag
- error_code
- affected_business_id

- 破坏性变更必须升级大版本
- 废弃接口保留过渡期
- 文档版本与代码版本同步
