# 销售采购一体化智能体平台代码工程

本工程基于前述 10 份企业级方案文档开始落地实现，目标是构建生产级、可持续扩展的销售 + 采购一体化智能体平台。

## 工程结构

- `apps/api`：FastAPI 后端服务
- `apps/web`：React + TypeScript 前端工作台
- `packages`：后续可沉淀共享包

## 第一阶段已完成

### 后端
- 企业级配置中心
- CORS、日志、异常处理
- SQLAlchemy 基础数据库层
- Alembic 基础迁移骨架
- 四大业务域模型骨架
- 销售、方案、供应商、采购、知识、AI、审计 API 骨架
- 领域服务与编排服务初始实现
- Dockerfile、测试样例、seed 脚本

### 前端
- React + Vite + TypeScript 工程
- 企业级工作台主布局
- 首页工作台
- 销售线索页面
- 方案页面
- 采购页面
- 供应商页面
- 知识中心、运营中心、设置中心
- 基础组件与全局样式

## 第二阶段当前已推进

### 后端新增
- 用户、角色、权限模型
- 密码加密工具
- 用户服务与登录逻辑
- 用户列表 / 创建用户接口
- 线索创建、更新、详情、跟进记录接口
- 方案创建与详情接口
- 供应商创建与详情接口
- 采购申请创建接口
- 更接近真实 CRUD 的数据库访问逻辑

### 前端新增
- 销售线索详情卡片
- 销售线索页支持选中详情查看
- 设置中心接入用户列表
- 页面样式补强，支持详情块与按钮操作

## 当前实现定位

当前版本已经进入“第二阶段：真实业务基础能力”的开发中，不再只是页面和接口空壳，而是在向可录入、可查询、可操作、可鉴权的方向推进。
